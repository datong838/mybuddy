# 故障话术（试用现场）

| 现象 | 可能原因 | 话术 / 动作 |
| --- | --- | --- |
| 打不开控制台 | Docker 未起 / 端口占用 | 「我们先运行健康检查脚本，确认 Docker 与 80 端口。」 |
| Postgres 反复重启（WSL） | 数据目录在 `/mnt/c` | 确认使用 `docker-compose.wsl-volumes.yaml` |
| 顶栏红灯 | 知识引擎/接入服务不可达 | 「请点刷新健康；按修复说明启动 Docker 与本机服务。」 |
| 答案空 / error | 无应用密钥或模型失败 | 「请配置模型并填入本机 .env 应用访问密钥。」 |
| **健康绿仍答不了** · `Provider openai does not exist` | **知识检索节点**仍为「单路召回」，隐藏的 `single_retrieval_config` 指向 `openai/gpt-3.5-turbo`（与 LLM 节点无关） | 「健康灯只表示服务可达。工作室→编排→点**知识检索**→**召回设置**→把「系统推理模型」改成已配置模型，或切到 N 路召回并关 Rerank，**保存并发布**后再问。」 |
| 知识检索只显示 1 个库，DB 仍有已删库 UUID | 加载时 UI 只画存在库，却把原 `dataset_ids`（含幽灵）原样写回 | 非操作失误。见 `10f`；用 `scripts/win/fix-ghost-dataset-ids.sql` 清数据，或重建前端使裁剪补丁生效 |
| 顶栏有详情/refuse_reason | 接入服务已回传上游原因 | 按详情文案改编排或 `.env` Key |
| Web 聊天页见 POWERED BY | 工作区未去品牌 | 见 `10g`；执行 `fix-remove-webapp-brand.sql` 或工作室自定义品牌关闭 |
| 无引用 | 未上传语料 / TopK | 「请上传 sample-corpus 并重建索引。」 |
| 杀软拦截 | 安装包误报 | 「可先解压便携目录运行；签名二期加固。」 |
| 内存不足 | &lt;16GB | 「建议关掉无关应用或升级到 16GB。」 |
| OpenOcta 起不来 | 未装二进制 | 「0.1 主路径可用 adapter 直连；OpenOcta 见起服说明。」 |

卸载：停 compose → 删命名卷（注意数据）→ 卸桌面客户端。
