# 试用说明（售前 / 交付可读）

## 产品是什么

**AOS 本地工作站（试用版）**：本机三栏桌面（对象 · Chat · 引用溯源）+ 本机知识引擎（文档检索与编排）。  
数据默认停留在客户指定电脑（`127.0.0.1`），**不是**钉钉/企微/飞书机器人，也不是完整 Ontology/Workshop。

## 启动 / 关闭（命令行）

工作目录：`mybuddy-v01`。完整表见仓库方案 [10d](../../docs/palantier/10_v01/10d-v0.1-Phase4收口说明.md)。

```powershell
# 启动（Docker + 知识引擎 + 接入服务）
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\win\start-all.ps1

# 桌面
.\dist\windows\AOS-Local-Workstation.exe

# 关闭（接入服务 + 知识引擎）
powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\win\stop-all.ps1
```

macOS / Linux：`bash ./scripts/mac/start-all.sh` · `stop-all.sh`（linux 同理）。

## 15 分钟体验路径

1. 按检查单确认 Docker / 内存。  
2. 执行 `start-all` → 打开本机控制台完成知识引擎初始化。  
3. 上传 `docs/sample-corpus/`（或客户文档）并配置模型。  
4. 正式试用写入应用访问密钥（`.env`）后重启接入服务（或重新 `start-all`）。  
5. 打开桌面客户端（安装包或 `npm run tauri dev`）。  
6. 左栏点对象 → 中栏提问 → 右栏看引用。  
7. 按 `scripts/selftest/run-golden-checklist.md` 勾选 ≥10 题。  
8. 结束试用执行 `stop-all`。

无模型 Key 时可用演示态：`BUDDY_DEMO_MODE=1`（`start-all` 无 Key 时自动开启；仅演示契约与 UI，非正式试用结论）。
