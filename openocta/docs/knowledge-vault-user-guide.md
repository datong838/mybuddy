# 知识库（Knowledge Vault）用户使用手册

> **文档版本**：UI 工作台版（2026-07）  
> **适用产品**：OpenOcta / DataBuff 桌面端 · 顶栏「知识库」标签页  
> **前置条件**：Gateway 已连接  
> **延伸阅读**：[知识库产品说明](./knowledge-vault.md)（架构、Agent 工具、配置项）

---

## 1. 知识库是什么

知识库是一套 **Obsidian 风格的 Markdown 笔记工作台**：你在本地维护 `.md` 笔记，在 OpenOcta 界面中浏览、编辑、检索；对话中的 Agent 则通过 **`memory_search`** 工具检索同一份索引，回答 Runbook、架构说明等问题。

| 你能做什么 | Agent 能做什么 |
|------------|----------------|
| 在界面新建/编辑/删除笔记与文件夹 | 用自然语言检索 Vault 内容 |
| 全文搜索、查看文档关系图谱 | 返回路径、行号、摘要片段 |
| 点击「同步索引」更新检索库 | 结合 bash、MCP 等工具执行任务 |

## 2. 界面总览

```
┌─────────────────────────────────────────────────────────────┐
│  知识库    [配置目录]  [文档|图谱]  [刷新]  [同步索引]         │
├──────────────┬──────────────────────────────────────────────┤
│  新建笔记     │  面包屑 / 编辑|预览|分屏  [保存]              │
│  新建文件夹   │                                              │
│  ─────────   │         笔记正文（编辑或预览）                 │
│  [文件名|全文]│                                              │
│  搜索框       │                                              │
│  文件树       │                                              │
│  (可拖拽调宽) │                                              │
├──────────────┴──────────────────────────────────────────────┤
│  Vault 路径 · N 篇笔记 · M 个索引块 · 上次同步时间            │
└─────────────────────────────────────────────────────────────┘
```

### 2.1 顶栏按钮

| 按钮 | 作用 |
|------|------|
| **配置目录** | 修改 Vault 根目录（绝对路径）；留空则恢复默认路径 |
| **文档** | 左侧文件树 + 右侧编辑区（默认） |
| **图谱** | 笔记之间的双链 / Markdown 链接关系图 |
| **刷新** | 重新加载文件列表；若当前有打开的笔记则一并刷新内容 |
| **同步索引** | 重建检索索引，供 Agent `memory_search` 使用 |

### 2.2 底栏状态条

| 信息 | 含义 |
|------|------|
| **Vault 路径** | 当前知识库根目录 |
| **N 篇笔记** | 已索引的 Markdown 文件数量 |
| **M 个索引块** | 分块索引数量（长文会被切成多块） |
| **上次同步** | 最近一次成功同步索引的时间 |

## 3. 快速上手（5 分钟）

1. 打开顶栏 **「知识库」**，确认底栏已显示 Vault 路径（默认多为 `<workspace>/vault` 或 `~/.openocta/vault`）。
2. 点击 **「新建笔记」**，输入名称、选择父级目录（默认根目录），创建第一篇笔记。
3. 在右侧选择 **「编辑」**，写入 Markdown 内容，按 **`Ctrl+S`**（Mac：`⌘+S`）保存。
4. 点击 **「同步索引」**，等待底栏索引块数更新。
5. 回到 **对话** 页，向 Agent 提问与笔记相关的内容；Agent 应通过 `memory_search` 检索后再回答。

> **重要**：在界面中改笔记后，务必 **保存** 并 **同步索引**；若 Agent 仍引用旧内容，请在会话中 **再发一条新消息** 触发重新检索。

## 4. 文件树与笔记管理

### 4.1 浏览与选择

- **点击笔记**：在中间区域打开该笔记。
- **点击文件夹**：展开 / 折叠；文件夹行会高亮为「选中」状态。
- **侧栏宽度**：拖拽侧栏右边缘可调宽（约 200–520px），宽度会自动记住。

### 4.2 新建笔记 / 文件夹

1. 点击侧栏上方 **「新建笔记」** 或 **「新建文件夹」**。
2. 在弹窗中填写：
   - **名称**：笔记无需写 `.md` 后缀（系统自动补全）。
   - **父级目录**：下拉选择，默认 **根目录**；若当前选中了某篇笔记或文件夹，会默认选中其所在目录。
3. 点击 **「创建」**。

**同名校验**：若同目录下已有同名笔记或文件夹，弹窗内会显示红色提示：「目标路径已存在，请更换名称或父级目录」。

### 4.3 编辑与删除（行内按钮）

将鼠标悬停在文件树某一行，或选中该行后，右侧会出现两个图标：

| 图标 | 功能 |
|------|------|
| ✏️ **编辑** | 打开弹窗：可 **重命名**、**更换父级目录**（移动） |
| 🗑️ **删除** | 删除该笔记或文件夹 |

**删除文件夹**：

- 空文件夹：确认后直接删除。
- 非空文件夹：第一次会提示失败；再次确认可 **强制删除** 文件夹及其中全部内容。

### 4.4 在 Obsidian 中编辑（可选）

知识库本质是磁盘上的普通文件夹。你也可以：

1. 用 Obsidian **「Open folder as vault」** 打开同一目录；
2. 在 Obsidian 中编辑保存；
3. 回到 OpenOcta 点击 **「刷新」** 与 **「同步索引」**。

`.obsidian/` 配置目录不会被索引，不影响检索。

## 5. 搜索

侧栏搜索框上方有两种模式：

### 5.1 文件名（默认）

- 输入即 **过滤** 左侧文件树中的笔记名称与路径。
- 匹配的笔记会自动展开所在文件夹。

### 5.2 全文

- 输入关键词后按 **`Enter`** 触发检索（与 Agent `memory_search` **同源索引**）。
- 结果列表显示：标题、路径、行号、内容摘要。
- **点击某条结果**：打开对应笔记，并尝试滚动到匹配行附近。

---

## 6. 阅读与编辑笔记

### 6.1 三种视图

工具栏提供三种模式：

| 模式 | 说明 |
|------|------|
| **编辑** | 纯文本 Markdown 编辑区，左侧带行号 |
| **预览** | 渲染后的 Markdown 阅读视图 |
| **分屏** | 左编辑、右实时预览 |

### 6.2 面包屑

笔记顶部显示路径，例如 `notes / 2024 / 部署手册`。点击文件夹段可在左侧树中展开对应目录。

### 6.3 未保存提示

- 有未保存修改时，工具栏会出现 **橙色圆点**。
- 切换笔记或关闭页面前会弹出确认。
- 保存成功后圆点消失。

### 6.4 快捷键

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+S` / `⌘+S` | 保存当前笔记 |
| `Ctrl+B` / `⌘+B` | 为选中文字加粗 `**...**`（编辑模式且光标在编辑区） |
| `Ctrl+I` / `⌘+I` | 为选中文字斜体 `*...*` |

工具栏也提供 **B**、**I** 按钮与 **保存** 按钮。

### 6.5 链接语法（供图谱与 Agent 使用）

| 写法 | 用途 |
|------|------|
| `[[另一篇笔记]]` | Obsidian 双链；会出现在 **图谱** 中 |
| `[标题](path/to/note.md)` | Markdown 链接；同样计入图谱 |

保存后建议 **同步索引**，以便 Agent 检索到新内容。

## 7. 文档关系图谱

1. 顶栏切换到 **「图谱」**。
2. **滚轮**：以视口中心缩放；**拖拽空白处**：平移画布。
3. **悬停节点**：显示完整标题；放大至约 150% 以上时显示完整标签。
4. **点击节点**：自动切回 **「文档」** 视图并打开该笔记（即使当前已选中同一笔记）。

图谱基于笔记内的双链与 Markdown 链接生成；无链接的笔记显示为孤立节点。

## 8. 同步索引与 Agent 检索

### 8.1 何时需要「同步索引」

| 操作 | 建议 |
|------|------|
| 在 OpenOcta 界面保存笔记 | 保存后会 **自动触发同步**（可留意顶栏绿色提示） |
| 在 Obsidian / 外部编辑器改文件 | 回到知识库点 **刷新** + **同步索引** |
| 新建 / 删除 / 移动笔记 | 完成后点 **同步索引** |
| Agent 搜不到刚写的内容 | 先同步索引，再在对话里 **新发一条消息** |

### 8.2 Agent 如何检索

Agent 使用工具 **`memory_search`**（不是自动读整个 Vault）。检索结果包含：

- 笔记相对路径 `path`
- 行号范围 `start_line` / `end_line`
- 文本摘要 `snippet`

这与界面 **「全文」搜索** 使用同一套索引。

### 8.3 会话内追问

与 Vault 无关、仅针对 **当前对话** 的「刚才说的…」类问题，Agent 会使用 **`session_search`** 检索会话历史。详见 [产品说明 · Agent 检索工具](./knowledge-vault.md#七agent-检索工具)。

## 9. 配置 Vault 目录

1. 点击顶栏 **「配置目录」**。
2. 输入 **绝对路径** 作为新的 Vault 根目录。
3. **留空并确认** 可恢复为默认路径（`workspace/vault` 或 `~/.openocta/vault`）。
4. 修改后会自动刷新列表并同步索引。

也可在 `openocta.json` 中配置：

```json
{
  "agents": {
    "defaults": {
      "knowledge": {
        "enabled": true,
        "vaultDir": "D:/notes/my-vault"
      }
    }
  }
}
```

## 10. 常见问题

### Q1：Agent 说知识库里没有相关内容？

1. 确认笔记已 **保存** 且界面已 **同步索引**（看底栏块数与时间）。
2. 在知识库 **「全文」** 搜索同一关键词，确认 UI 能搜到。
3. 在对话中 **再发一条新消息**（不要只依赖旧上下文）。
4. 确认 `agents.defaults.knowledge.enabled` 未设为 `false`。

### Q2：改了 Obsidian 里的文件，界面没更新？

点击 **「刷新」**；若 Agent 要检索，再点 **「同步索引」**。

### Q3：新建时提示「目标路径已存在」？

同目录下已有同名笔记（`.md`）或同名文件夹。请换名称或换父级目录。

### Q4：图谱是空的或节点很少？

图谱只显示 **笔记之间的链接**。请检查是否使用了 `[[双链]]` 或 Markdown 链接；仅有文件夹层级不会在图谱中连线。

### Q5：可以存 PDF / Word 吗？

当前仅支持 **`.md` Markdown** 文件。

### Q6：索引文件要提交 Git 吗？

- **要提交**：`vault/` 下的 `.md` 笔记  
- **不要提交**：`~/.openocta/knowledge-index/`（本地自动重建）

### Q7：和「记忆 / Memory」插件配置冲突吗？

Knowledge Vault 由 `agents.defaults.knowledge` 控制，与顶层 `memory` 插件块 **相互独立**。详见 [产品说明 · 与其他记忆能力的关系](./knowledge-vault.md#九与其他记忆能力的关系)。

## 11. 操作速查表

| 我想… | 怎么做 |
|--------|--------|
| 新建笔记 | 侧栏「新建笔记」→ 填名称与父目录 → 创建 |
| 改笔记名或移动目录 | 文件树行 ✏️ → 改名称 / 父级目录 → 保存 |
| 删除笔记 | 文件树行 🗑️ → 确认 |
| 编辑正文 | 选笔记 →「编辑」或「分屏」→ 修改 → `Ctrl+S` |
| 按文件名找 | 搜索选「文件名」→ 输入过滤 |
| 按正文找 | 搜索选「全文」→ 输入 → `Enter` |
| 看笔记关系 | 顶栏「图谱」 |
| 从图谱打开笔记 | 点击节点（自动回文档视图） |
| 让 Agent 能搜到新内容 | 保存 →「同步索引」→ 对话新发消息 |
| 换 Vault 目录 |「配置目录」或改 `openocta.json` |
